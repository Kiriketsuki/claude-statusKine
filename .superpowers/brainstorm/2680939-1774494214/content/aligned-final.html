Account Detection Fix

Bug: Current account detection uses workspace path (*/workdev/Aurrigo* → Aurrigo). If you switch accounts mid-session or use personal account in an Aurrigo workspace, usage cache shows wrong account's data.

Fix: Detect account from the status JSON itself. Claude Code provides .session or account info in the status payload. Alternatively, read $CLAUDE_CONFIG_DIR if set, or check the active credentials file identity. The cache file key should be derived from the actual authenticated account, not the workspace path.

Impact: Affects fetch-usage.sh (which account's API to poll), statusline-command.sh (which cache file to read), and the email display on L1.

Alignment Strategy

Column alignment across L2/L3:
- L2 and L3 both start with ▰ XXX [8-bar] XX% — these form a natural alignment column
- Use %3d%% format for percentages (right-justified, 3 chars) so  5% and 90% align
- Token values in L3 right-justified within each pair: c⧈ 120k / c⧈  60k
- Bridge starts at the same column on L2/L3/L4 when possible (flex distributes remaining space)

Replaces the current first_w / l2_5h_pad hack with a general max-width-per-column pass across all metric lines (L2, L3, L4).
