Statusline Animation Research

What we found across docs, GitHub, and the community

The Core Constraint

Claude Code Statusline Refresh Triggers (Official Docs)
1 After each new assistant message
2 When permission mode changes
3 When vim mode toggles
~ Debounced at 300ms (rapid changes batch)
No timer/interval/cron support. The statusline is strictly event-driven. There is no refreshIntervalSeconds config key.

What Other Statuslines Do

ccstatusline (sirmalloc) — No animation. Static renders with powerline glyphs. TUI config. 2.2k+ npm downloads.
CCometixLine (Haleclipse) — Rust-based. No animation. Focus on performance.
claude-code-statusline (levz0r) — No animation. Token tracking + cost.
Yours (Chrysaki) — The only one with time-based animation logic (grad_phase, wave_shift, badge_tick, pulse_scale, bridge_rot). But it only advances between renders.

Bonus Discovery: Native Rate Limits

Claude Code now sends rate limit data natively in the statusline JSON!
rate_limits.five_hour.used_percentage (0-100)
rate_limits.seven_day.used_percentage (0-100)
rate_limits.five_hour.resets_at (Unix epoch seconds)
rate_limits.seven_day.resets_at (Unix epoch seconds)
This could replace your fetch-usage.sh entirely — no more OAuth token caching, background fetching, or credential file parsing.
