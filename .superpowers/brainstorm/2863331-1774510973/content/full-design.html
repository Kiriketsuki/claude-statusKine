Full Design: Statusline v2.1

Three workstreams -- animation prep, native rate limits, visual polish

Workstream 1: Animation System (future-ready)

Design the phase system so it animates smoothly when refreshIntervalSeconds lands. Until then, each render picks a visually distinct static frame.

Keep all existing phase variables (grad_phase, wave_shift, badge_tick, pulse_scale, _bridge_rot)
They still drive wave bars, badge pulse, and bridge colour rotation. Only gradient rendering is removed -- the phase math stays.
Add _jewel_seed -- timestamp-derived seed for the 9-colour jewel pool
_jewel_seed=$(date +%s)
Each line picks its colour via: JEWEL_COLORS[(_jewel_seed / P) % 9] where P is a different prime per line (1, 3, 7, 11). This ensures colours shift on every render and no two adjacent lines share a colour.
Remove gradient_text() and gradient_text_off()
~65 lines of per-character colour interpolation code deleted. Replaced by simple printf "%b%s%b" "$colour" "$text" "$R" calls.
Future: re-add gradient as an animation mode
When refreshIntervalSeconds ships, add back gradient rendering behind a CHRYSAKI_ANIMATE=1 flag. The phase math is already there -- just wire it to a new animated_gradient_text() that only activates at >=2fps refresh.

Workstream 3: Migrate to Native Rate Limits

Claude Code now sends rate limit data in the statusline JSON. Drop the background fetcher.

Data Source Migration
OLD: fetch-usage.sh → OAuth API → /tmp/.claude_usage_cache
NEW: JSON stdin → rate_limits.five_hour / rate_limits.seven_day
Fields available:
rate_limits.five_hour.used_percentage → replaces five_h (0-100)
rate_limits.five_hour.resets_at → replaces five_h_reset (Unix epoch)
rate_limits.seven_day.used_percentage → replaces seven_d (0-100)
rate_limits.seven_day.resets_at → replaces seven_d_reset (Unix epoch)
Delete:
  • fetch-usage.sh -- entire file
  • OAuth token caching logic (/tmp/.claude_token_cache)
  • Per-account usage cache (/tmp/.claude_usage_cache_*)
  • Background fetch trigger in statusline-command.sh (line 115)
  • Account config dir resolution for usage cache (lines 94-116)
Replace with:
five_h=$(echo "$input" | jq -r '.rate_limits.five_hour.used_percentage // empty' | cut -d. -f1)
seven_d=$(echo "$input" | jq -r '.rate_limits.seven_day.used_percentage // empty' | cut -d. -f1)
five_h_reset_epoch=$(echo "$input" | jq -r '.rate_limits.five_hour.resets_at // empty')
seven_d_reset_epoch=$(echo "$input" | jq -r '.rate_limits.seven_day.resets_at // empty')
Simplify compute_delta(): receives Unix epoch directly instead of ISO 8601. No more date parsing -- just diff=$(( reset_epoch - now_epoch )).
Keep account config resolution for: email display and fetch-stats.sh (issues/PR). Only the usage-specific code is removed.
